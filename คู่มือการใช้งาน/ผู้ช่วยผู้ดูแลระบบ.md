# คู่มือการใช้งานระบบ SRI-KETGUIDE สำหรับผู้ช่วยผู้ดูแลระบบ (Staff)

**บทบาท:** Staff (ผู้ช่วยผู้ดูแลระบบ)  
**หน้าที่หลัก:** ตรวจสอบและอนุมัติสลิปการชำระเงินของผู้ลงโฆษณา (Sponsor)

⚠️ **หมายเหตุ:** Staff มีหน้าที่หลักในการตรวจสอบและอนุมัติสลิปการชำระเงินเท่านั้น การจัดการอื่นๆ (เช่น จัดการผู้ใช้, แพ็กเกจ) เป็นหน้าที่ของ Admin เท่านั้น

## 1. การเข้าสู่ระบบ

### 1.1 หน้าล็อกอิน

```
+-------------------------------------------------------------------+
|                                                                   |
|                     SRI-KETGUIDE                                  |
|           ระบบประชาสัมพันธ์การท่องเที่ยวอัจฉริยะ                    |
|                                                                   |
|  +----------------------------------------------------------+     |
|  |                                                          |     |
|  |   อีเมล: [____________________________]                   |     |
|  |                                                          |     |
|  |   รหัสผ่าน: [____________________________]                |     |
|  |                                                          |     |
|  |              [  เข้าสู่ระบบ  ]                             |     |
|  |                                                          |     |
|  |   [ ] จดจำการเข้าสู่ระบบ                                   |     |
|  |                                                          |     |
|  |   ลืมรหัสผ่าน?                                          |     |
|  |                                                          |     |
|  +----------------------------------------------------------+     |
|                                                                   |
|  URL: https://frontend.sri-ketguide.com/#/login                |
|                                                                   |
+-------------------------------------------------------------------+
```

### 1.2 วิธีการเข้าสู่ระบบ

1. เปิดเว็บเบราว์เซอร์และไปที่ URL: `https://frontend.sri-ketguide.com/#/login`
2. กรอกข้อมูลเข้าสู่ระบบ:
   - **อีเมล**: กรอกอีเมลที่ได้จากผู้ดูแลระบบ
   - **รหัสผ่าน**: กรอกรหัสผ่านที่ได้รับ
3. คลิกปุ่ม **"เข้าสู่ระบบ"** หรือกด Enter
4. หากต้องการให้ระบบจำการเข้าสู่ระบบ ให้เลือก **"จดจำการเข้าสู่ระบบ"**

### 1.3 บัญชีผู้ใช้งานสำหรับ Staff

**บัญชี Staff:**
- อีเมล: `staff@example.com`
- รหัสผ่าน: `1234`
- สิทธิ์: `staff`
- หน้าที่: ตรวจสอบและอนุมัติสลิปการชำระเงิน

⚠️ **หมายเหตุ:** เนื่องจากบทบาท Staff มีหน้าที่หลักในการอนุมัติสลิปเท่านั้น คู่มือนี้จะเน้นที่การใช้งานส่วนนั้นเป็นหลัก

### 1.4 ข้อควรระวังในการล็อกอิน

⚠️ **คำเตือน:**
- อย่าแชร์ข้อมูลการเข้าสู่ระบบกับผู้อื่น
- ออกจากระบบเมื่อเสร็จสิ้นการทำงาน
- ไม่ควรเลือก "จดจำการเข้าสู่ระบบ" เมื่อใช้คอมพิวเตอร์สาธารณะ
- หากลืมรหัสผ่าน ให้ติดต่อผู้ดูแลระบบหลัก

### 2.2 องค์ประกอบของหน้า Staff Panel

หน้าหลัก Staff Panel ประกอบด้วย:

1. **แถบนำทาง (Navigation Bar)**
   - Logo และชื่อระบบ
   - เมนู: Chat, Staff Panel, Sponsor Panel, Admin Panel
   - ช่องค้นหา
   - ปุ่มออกจากระบบ

2. **พื้นที่แสดงข้อมูล (Content Area)**
   - แสดงสลิปการชำระเงินที่รอการตรวจสอบ
   - ตารางสลิปพร้อมปุ่มจัดการ (ดู, อนุมัติ, ปฏิเสธ)

3. **ตัวกรอง (Filter)**
   - กรองตามสถานะ: ทั้งหมด, รอการตรวจสอบ, อนุมัติแล้ว, ปฏิเสธ

### 2.3 ความแตกต่างระหว่าง Staff และ Admin

⚠️ **สิทธิ์ของ Staff:**
- ✅ ดูสลิปการชำระเงินทั้งหมด
- ✅ อนุมัติสลิปการชำระเงิน
- ✅ ปฏิเสธสลิปการชำระเงิน
- ❌ จัดการผู้ใช้ (Admin เท่านั้น)
- ❌ จัดการแพ็กเกจ (Admin เท่านั้น)
- ❌ จัดการธุรกิจ (Admin เท่านั้น - แต่ในบางระบบ Staff อาจทำได้)

## 4. การดูและตรวจสอบสลิป

### 4.1 การดูสลิป

**ขั้นตอน:**

1. หาสลิปที่ต้องการดู
2. คลิกปุ่ม **"View"** (ไอคอน 🖼️)
3. ระบบจะแสดงภาพสลิป:

```
+-------------------------------------------------------------------+
|                    ภาพสลิปการชำระเงิน                              |
|  ┌─────────────────────────────────────────────────────────┐     |
|  │                                                         │     |
|  │                  [ภาพสลิปขนาดใหญ่]                        │     |
|  │                                                         │     |
|  │  รหัสสลิป: #1a2b3c4d                                    │     |
|  │  ธุรกิจ: บาบาราย โฮเทล                                   │     |
|  │  แพ็กเกจ: Premium                                        │     |
|  │  จำนวนเงิน: 999 บาท                                      │     |
|  │  วันที่: 1 มกราคม 2568                                    │     |
|  │                                                         │     |
|  │              [ปิด]                                      │     |
|  │                                                         │     |
|  └─────────────────────────────────────────────────────────┘     |
+-------------------------------------------------------------------+
```

**Path API:**
- `GET /api/payment-slips/{slipId}`
- Path รูปภาพ: `/uploads/payment-slips/{filename}`

## 5. การอนุมัติสลิป

**ขั้นตอน:**

1. หาสลิปที่มีสถานะ **"รอการตรวจสอบ"** (⏳)
2. คลิกปุ่ม **"Approve"** (ไอคอน ✓)
3. ระบบจะแสดงข้อความยืนยัน:

```
+-------------------------------------------------------------------+
|                    ยืนยันการอนุมัติ                                |
|                                                                   |
|  คุณต้องการอนุมัติสลิป #1a2b3c4d หรือไม่?                         |
|                                                                   |
|  ธุรกิจ: บาบาราย โฮเทล                                            |
|  แพ็กเกจ: Premium                                                |
|  จำนวนเงิน: 999 บาท                                               |
|                                                                   |
|           [ยกเลิก]              [อนุมัติ]                          |
+-------------------------------------------------------------------+
```

4. คลิก **"อนุมัติ"** เพื่อยืนยัน

**Path API:**
- `POST /api/payment-slips/{slipId}/approve`
- ต้องมี Authorization Header และ Role: `staff` หรือ `admin`

**ผลลัพธ์:**
- สถานะสลิปจะเปลี่ยนเป็น **"อนุมัติแล้ว"** (✅)
- ธุรกิจจะได้รับแพ็กเกจตามที่ชำระเงิน
- ระบบจะสร้างหรืออัปเดต Business Package ให้อัตโนมัติ

---

## 6. การปฏิเสธสลิป

**ขั้นตอน:**

1. หาสลิปที่มีสถานะ **"รอการตรวจสอบ"**
2. คลิกปุ่ม **"Reject"** (ไอคอน ✗)
3. กรอกเหตุผลการปฏิเสธ (ถ้ามี):

```
+-------------------------------------------------------------------+
|                    ปฏิเสธสลิป                                     |
|                                                                   |
|  คุณต้องการปฏิเสธสลิป #1a2b3c4d หรือไม่?                         |
|                                                                   |
|  เหตุผลการปฏิเสธ (ไม่บังคับ):                                      |
|  [_________________________________________]                     |
|  [_________________________________________]                     |
|                                                                   |
|           [ยกเลิก]              [ปฏิเสธ]                          |
+-------------------------------------------------------------------+
```

4. คลิก **"ปฏิเสธ"** เพื่อยืนยัน

**Path API:**
- `POST /api/payment-slips/{slipId}/reject`
- ต้องมี Authorization Header และ Role: `staff` หรือ `admin`

**ผลลัพธ์:**
- สถานะสลิปจะเปลี่ยนเป็น **"ปฏิเสธ"** (❌)
- ธุรกิจจะได้รับการแจ้งเตือน (ถ้ามีระบบแจ้งเตือน)
- ธุรกิจสามารถอัปโหลดสลิปใหม่ได้

## 7. การกรองและค้นหาสลิป

### 7.1 การกรองสลิป

**ขั้นตอน:**

1. คลิกที่ dropdown **"ทั้งหมด"** ที่มุมขวาบน
2. เลือกสถานะที่ต้องการ:
   - **ทั้งหมด**: แสดงทุกสลิป
   - **รอการตรวจสอบ**: แสดงเฉพาะสลิปที่รออนุมัติ
   - **อนุมัติแล้ว**: แสดงเฉพาะสลิปที่อนุมัติแล้ว
   - **ปฏิเสธ**: แสดงเฉพาะสลิปที่ปฏิเสธ

**Path API:**
- `GET /api/payment-slips?status={status}`

⚠️ **คำเตือน:**
- ควรตรวจสอบสลิปอย่างละเอียดก่อนอนุมัติ
- ตรวจสอบว่า:
  - จำนวนเงินตรงกับแพ็กเกจที่เลือก
  - วันที่และเวลาการโอนถูกต้อง
  - ข้อมูลธุรกิจถูกต้อง
  - ภาพสลิปชัดเจนและอ่านได้

## 8. สิทธิ์และข้อจำกัดของ Staff

### 8.1 สิ่งที่ Staff ทำได้

✅ **สามารถทำได้:**
- ดูสลิปการชำระเงินทั้งหมด
- ดูรายละเอียดสลิปแต่ละรายการ
- อนุมัติสลิปการชำระเงิน
- ปฏิเสธสลิปการชำระเงิน
- กรองสลิปตามสถานะ

### 8.2 สิ่งที่ Staff ไม่สามารถทำได้

❌ **ไม่สามารถทำได้:**
- ❌ จัดการผู้ใช้ (สร้าง, แก้ไข, ลบ, ระงับ) - Admin เท่านั้น
- ❌ จัดการแพ็กเกจ (สร้าง, แก้ไข, ลบ) - Admin เท่านั้น
- ❌ จัดการธุรกิจ (ในบางกรณี) - อ้างอิงจากสิทธิ์จริงของระบบ
- ❌ ตั้งค่าระบบ

### 8.3 เมื่อต้องการฟีเจอร์เพิ่มเติม

หาก Staff ต้องการจัดการส่วนอื่นๆ:
1. ติดต่อ Admin เพื่อขอสิทธิ์เพิ่มเติม
2. หรือขอให้ Admin ทำการจัดการแทน
3. หรือขออัปเกรดเป็น Admin (ขึ้นอยู่กับการตั้งค่าของระบบ)

⚠️ **หมายเหตุ:** การจัดการส่วนอื่นๆ ควรทำผ่าน Admin เท่านั้นเพื่อความปลอดภัยและการควบคุมที่เหมาะสม

## 9. คำถามที่พบบ่อย (FAQ)

### 9.1 คำถามเกี่ยวกับสิทธิ์ของ Staff

**Q: Staff สามารถจัดการผู้ใช้หรือแพ็กเกจได้หรือไม่?**
A: ❌ **ไม่ได้** Staff สามารถทำได้แค่อนุมัติและปฏิเสธสลิปการชำระเงินเท่านั้น การจัดการผู้ใช้และแพ็กเกจเป็นหน้าที่ของ Admin

**Q: ฉันต้องการสิทธิ์เพิ่มเติมเพื่อจัดการธุรกิจ ควรทำอย่างไร?**
A: ติดต่อ Admin เพื่อ:
- ขอเพิ่มสิทธิ์ให้กับบัญชี Staff ของท่าน
- หรือขออัปเกรดเป็น Admin
- หรือขอให้ Admin ทำการจัดการแทน

### 9.2 คำถามเกี่ยวกับการอนุมัติสลิป

**Q: สลิปที่อนุมัติไปแล้วสามารถยกเลิกได้หรือไม่?**
A: ไม่สามารถยกเลิกได้โดยตรง ควรติดต่อ Admin เพื่อดำเนินการถ้าเกิดข้อผิดพลาด

**Q: ฉันเห็นสลิปที่ผิดปกติ ควรทำอย่างไร?**
A: 
1. ปฏิเสธสลิปและบันทึกเหตุผลให้ชัดเจน
2. หากไม่แน่ใจ ควรติดต่อ Admin เพื่อขอคำแนะนำ
3. ไม่ควรอนุมัติถ้ามีความสงสัย

**Q: ควรตรวจสอบอะไรก่อนอนุมัติสลิป?**
A: ตรวจสอบ:
- ✅ จำนวนเงินตรงกับแพ็กเกจที่เลือก (Premium: 999 บาท, Standard: 599 บาท, Free: 0 บาท)
- ✅ วันที่และเวลาการโอนถูกต้อง
- ✅ ข้อมูลธุรกิจตรงกับสลิป (ชื่อธุรกิจ, เจ้าของ)
- ✅ ภาพสลิปชัดเจนและอ่านได้

**Q: ถ้าสลิปถูกปฏิเสธ ผู้ลงโฆษณาจะทำอย่างไร?**
A: ผู้ลงโฆษณาสามารถอัปโหลดสลิปใหม่ได้ แต่ควรตรวจสอบให้แน่ใจว่าสลิปใหม่ถูกต้องก่อนส่ง

## 10. คำสั่งและ Path ที่เกี่ยวข้อง

### 10.1 API Endpoints สำหรับ Staff

#### 10.1.1 สลิปการชำระเงิน (Payment Slips) - หน้าที่หลัก

```
GET    /api/payment-slips                 - ดึงรายการสลิปทั้งหมด (Staff/Admin/Sponsor)
GET    /api/payment-slips/:id             - ดึงข้อมูลสลิปรายละเอียด (Staff/Admin)
POST   /api/payment-slips/:id/approve     - อนุมัติสลิป (Staff/Admin) ⭐
POST   /api/payment-slips/:id/reject      - ปฏิเสธสลิป (Staff/Admin) ⭐
```

**ตัวอย่าง Request (อนุมัติ):**
```bash
POST /api/payment-slips/123/approve
Authorization: Bearer {accessToken}
```

**ตัวอย่าง Request (ปฏิเสธ):**
```bash
POST /api/payment-slips/123/reject
Authorization: Bearer {accessToken}
Content-Type: application/json

{
  "reason": "จำนวนเงินไม่ตรงกับแพ็กเกจ"
}
```

#### 10.1.2 API อื่นๆ ที่ Staff ไม่สามารถใช้งานได้

❌ **ผู้ใช้ (Users)** - Admin เท่านั้น

❌ **Transaction** - ขึ้นอยู่กับการตั้งค่าของระบบ

### 10.4 คำสั่ง SQL ที่เกี่ยวข้องสำหรับ Staff

**คำสั่งที่ Staff อาจต้องการใช้:**

```sql
-- ดูสลิปที่รอการตรวจสอบ (หน้าที่หลัก)
SELECT id, business_id, status, amount, created_at 
FROM payment_slips 
WHERE status = 'pending'
ORDER BY created_at ASC;

-- ดูสลิปที่อนุมัติแล้ว
SELECT id, business_id, status, amount, created_at, approved_at
FROM payment_slips 
WHERE status = 'approved'
ORDER BY approved_at DESC
LIMIT 50;

-- ดูสลิปที่ปฏิเสธ
SELECT id, business_id, status, amount, created_at, rejected_at
FROM payment_slips 
WHERE status = 'rejected'
ORDER BY rejected_at DESC
LIMIT 50;

-- นับจำนวนสลิปตามสถานะ
SELECT status, COUNT(*) as count 
FROM payment_slips 
GROUP BY status;
```

⚠️ **หมายเหตุ:** 
- Staff ควรใช้ Web Interface เป็นหลัก
- คำสั่ง SQL เหล่านี้สำหรับการตรวจสอบข้อมูลเท่านั้น
- ไม่ควรแก้ไขข้อมูลใน Database โดยตรง (ควรใช้ Web Interface)

## 11. คำเตือนและข้อควรระวัง

### 11.1 การตรวจสอบสลิปการชำระเงิน

⚠️ **คำเตือน:**
1. **ตรวจสอบสลิปอย่างละเอียด**: 
   - จำนวนเงินต้องตรงกับแพ็กเกจ
   - วันที่และเวลาต้องถูกต้อง
   - ข้อมูลธุรกิจต้องตรงกับสลิป
2. **อนุมัติด้วยความระมัดระวัง**: การอนุมัติจะทำให้ธุรกิจได้รับแพ็กเกจทันที
3. **บันทึกเหตุผลการปฏิเสธ**: ควรบันทึกเหตุผลให้ชัดเจนเพื่ออ้างอิง

### 11.2 ความปลอดภัย

⚠️ **คำเตือน:**
1. **ออกจากระบบเมื่อเสร็จ**: ออกจากระบบทุกครั้งเมื่อเสร็จสิ้นการทำงาน
2. **ไม่แชร์ข้อมูลการเข้าสู่ระบบ**: อย่าแชร์อีเมลและรหัสผ่านกับผู้อื่น
3. **ใช้รหัสผ่านที่แข็งแรง**: ควรเปลี่ยนรหัสผ่านเป็นระยะๆ

### 11.3 ปัญหาที่พบบ่อยและวิธีแก้ไข

#### ปัญหา: ไม่สามารถเข้าสู่ระบบได้
**วิธีแก้ไข:**
1. ตรวจสอบอีเมล (`staff@example.com`) และรหัสผ่าน (`1234`)
2. ตรวจสอบว่าบัญชีถูกระงับหรือไม่ (ติดต่อ Admin)
3. ล้าง Cache และ Cookies ของเบราว์เซอร์
4. ติดต่อ Admin

#### ปัญหา: ไม่สามารถอนุมัติสลิปได้
**วิธีแก้ไข:**
1. ตรวจสอบว่ามีสิทธิ์ Staff
2. ตรวจสอบว่าสลิปอยู่ในสถานะ "รอการตรวจสอบ" หรือไม่
3. ตรวจสอบว่าไม่ได้อนุมัติไปแล้ว
4. ติดต่อ Admin ถ้ายังมีปัญหา

#### ปัญหา: สลิปไม่แสดง
**วิธีแก้ไข:**
1. ตรวจสอบตัวกรอง (Filter) - อาจกรองสถานะบางอย่างไว้
2. ลองรีเฟรชหน้าเว็บ
3. ตรวจสอบว่าไม่ถูกกรองตามวันที่
4. ติดต่อ Admin ถ้ายังไม่มีสลิปแสดง
